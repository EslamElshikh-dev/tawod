## طريقة التشغيل محليًا

المشروع Static Website ولا يحتاج إلى خطوة بناء أو تثبيت حزم.

### تشغيل مباشر

افتح ملف:

```text
index.html
```

داخل المتصفح مباشرة.

### تشغيل باستخدام Live Server

يفضل أثناء التطوير استخدام إضافة **Live Server** في VS Code:

1. افتح مجلد المشروع في VS Code.
2. اضغط بزر الفأرة الأيمن على `index.html`.
3. اختر **Open with Live Server**.
4. راجع الصفحات والروابط من المتصفح.

## النشر على GitHub Pages

المشروع مناسب للنشر المباشر عبر GitHub Pages.

### خطوات مقترحة

1. ارفع جميع ملفات المشروع في جذر المستودع.
2. تأكد من وجود ملف `index.html` في الجذر.
3. تأكد من وجود ملف `CNAME` ويحتوي على:

```text
tawodco.com
```

4. من إعدادات المستودع:
   - ادخل إلى **Settings**.
   - اختر **Pages**.
   - اختر النشر من فرع `main`.
   - اختر مجلد `/root` إن كان متاحًا.
5. تأكد من ضبط DNS الخاص بالنطاق `tawodco.com` حسب إعدادات GitHub Pages.
6. بعد النشر، اختبر الموقع والروابط وملف السايت ماب.

## إرشادات الصيانة

- عند إضافة صفحة جديدة، يجب إضافتها إلى `sitemap.xml`.
- عند تغيير رابط صفحة، يجب تحديث جميع الروابط الداخلية المرتبطة بها.
- عند إضافة صورة جديدة، يفضل استخدام WebP مع اسم ملف وصفي باللغة الإنجليزية.
- يجب كتابة Alt Text واضح للصور المهمة.
- يفضل الحفاظ على نفس نظام الألوان والخطوط والهوية البصرية.

- يجب مراجعة تجربة الجوال بعد أي تعديل في الهيدر، الفوتر، الكروت، أو أزرار التواصل.
